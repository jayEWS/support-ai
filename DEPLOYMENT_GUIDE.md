# Quick Deployment Guide for Enterprise AI Platform

## 🚀 **Method 1: VS Code Container Tools (Recommended for Development)**

### Prerequisites
- Install Docker Desktop on Windows
- VS Code with Docker extension (already installed)

### Steps
1. **Open in Container**:
   ```
   Ctrl+Shift+P → "Dev Containers: Reopen in Container"
   ```

2. **Use VS Code Tasks**:
   ```
   Ctrl+Shift+P → "Tasks: Run Task" → "🚀 Start Enterprise Stack"
   ```

3. **Monitor Services**:
   ```
   Ctrl+Shift+P → "Tasks: Run Task" → "🔍 Show Stack Status"
   ```

### Available Tasks
- `🐳 Build Enterprise Stack` - Build all containers
- `🚀 Start Enterprise Stack` - Start all services
- `🔍 Show Stack Status` - Check service health  
- `📊 Show Stack Logs` - View real-time logs
- `🛑 Stop Enterprise Stack` - Stop all services
- `☁️ Deploy to GCP VM` - Deploy to cloud

## ☁️ **Method 2: Direct GCP VM Deployment**

### Prerequisites
- GCP VM (Ubuntu 20.04+, minimum 4GB RAM)
- SSH access to your VM

### One-Command Deployment
```bash
# On your GCP VM:
curl -sSL https://raw.githubusercontent.com/jayEWS/support-edgeworks/main/deploy/gcp-deploy.sh | bash
```

### Manual Deployment Steps
1. **Copy deployment script**:
   ```bash
   scp -i ~/.ssh/id_ed25519 deploy/gcp-deploy.sh jay@34.126.104.27:/tmp/
   ```

2. **SSH to VM and run**:
   ```bash
   ssh -i ~/.ssh/id_ed25519 jay@34.126.104.27
   chmod +x /tmp/gcp-deploy.sh
   /tmp/gcp-deploy.sh
   ```

3. **Configure environment**:
   ```bash
   sudo nano /opt/support-ai/.env
   # Add your API keys (OPENAI_API_KEY, GOOGLE_GEMINI_API_KEY)
   ```

4. **Start services**:
   ```bash
   sudo systemctl start support-ai.service
   ```

## 📊 **Service Architecture**

| Service | Port | Purpose |
|---------|------|---------|
| **Nginx** | 80 | Reverse proxy & load balancer |
| **FastAPI** | Internal | AI orchestration layer |
| **Worker** | Internal | Background task processor |
| **Redis** | 6379 | Memory layer & message queue |
| **SQL Server** | 1433 | Relational data & episodic memory |
| **Qdrant** | 6333/6334 | Vector database & semantic memory |
| **Prometheus** | 9090 | Metrics collection |
| **Grafana** | 3000 | Monitoring dashboard |

## 🔧 **Production Configuration**

### Environment Variables (Required)
```bash
# AI Configuration
OPENAI_API_KEY=sk-your-openai-key
GOOGLE_GEMINI_API_KEY=your-gemini-key
LLM_PROVIDER=gemini

# Database
DATABASE_URL=mssql+pyodbc://sa:SecurePassword@db:1433/support_ai?driver=ODBC+Driver+18

# Security (Auto-generated by deployment script)
API_SECRET_KEY=auto-generated-secret
AUTH_SECRET_KEY=auto-generated-secret
```

### System Requirements
- **CPU**: 2+ cores
- **RAM**: 4GB+ (recommended: 8GB)
- **Storage**: 20GB+ SSD
- **Network**: Public IP with ports 80/443 open

## 🛠 **Management Commands**

### Docker Management
```bash
# View all containers
docker ps -a

# View specific service logs
docker logs support_portal_api -f

# Restart specific service
docker restart support_portal_api

# Scale worker processes
docker-compose -f docker-compose.enterprise.yml up -d --scale worker=3
```

### System Management
```bash
# Service status
sudo systemctl status support-ai.service

# Start/Stop/Restart
sudo systemctl start support-ai.service
sudo systemctl stop support-ai.service
sudo systemctl restart support-ai.service

# View system logs
journalctl -u support-ai.service -f
```

### Health Monitoring
```bash
# API health check
curl http://localhost/health

# Prometheus metrics
curl http://localhost:9090/api/v1/targets

# Qdrant status
curl http://localhost:6333/collections
```

## 🚨 **Troubleshooting**

### Common Issues
1. **Docker not found**: Install Docker Desktop on Windows
2. **Permission denied**: Add user to docker group on Linux
3. **Port conflicts**: Check no other services use ports 80, 3000, 9090
4. **Memory issues**: Ensure minimum 4GB RAM available
5. **Network issues**: Verify firewall allows ports 80/443

### Debug Commands
```bash
# Check container status
docker-compose -f docker-compose.enterprise.yml ps

# View all logs
docker-compose -f docker-compose.enterprise.yml logs

# Check system resources
docker stats

# Restart problematic service
docker-compose -f docker-compose.enterprise.yml restart api
```

## 🎯 **Next Steps After Deployment**

1. **Configure API Keys**: Update `.env` with your AI provider keys
2. **Setup Domain**: Point your domain to the VM IP
3. **Enable SSL**: Run `sudo certbot --nginx` for HTTPS
4. **Configure Monitoring**: Setup Grafana dashboards and alerts
5. **Backup Strategy**: Schedule database and vector store backups
6. **Security Hardening**: Configure firewall and fail2ban

**Access URLs:**
- Main App: `http://YOUR_VM_IP`
- Admin Panel: `http://YOUR_VM_IP/admin`
- Grafana: `http://YOUR_VM_IP:3000` (admin/admin)
- Prometheus: `http://YOUR_VM_IP:9090`
