Apa Ini?
--------
Ini adalah AI-powered Customer Support Portal untuk PT Edgeworks Solutions
Indonesia — perusahaan yang menjual sistem POS (Point of Sale) untuk restoran
dan retail. Portal ini menggantikan Freshdesk sebagai sistem tiket dan support
internal, dengan tambahan AI chatbot yang bisa menjawab pertanyaan customer
secara otomatis berdasarkan knowledge base produk Edgeworks.

                          FITUR UTAMA
================================================================================

1. AI CHATBOT (RAG-BASED)
--------------------------
- Customer buka web, chat, AI jawab otomatis dari knowledge base
- Pakai Hybrid Search: BM25 (keyword matching) + FAISS (semantic/vector search)
- Hasil digabung pakai Reciprocal Rank Fusion (RRF)
  - BM25 bobot 40%, Vector bobot 60%
- Confidence score dihitung
  - Kalau rendah, AI tetap coba jawab tapi dengan disclaimer
- Greeting detection — sapaan dijawab langsung tanpa cari KB
- Prompt bahasa Indonesia, nada ramah, panggil "Kak"


2. KNOWLEDGE BASE MANAGEMENT
-----------------------------
- Admin upload file apa saja (txt, pdf, excel, dll) lewat dashboard
- File di-chunk lalu di-embed pakai text-embedding-005
- Disimpan di FAISS index (lokal di disk)
- Source URL tracking per file
- Rebuild index otomatis setelah upload


3. TICKET SYSTEM
-----------------
- Auto-create tiket dari chat customer
- Status: Open, Pending, Closed, Resolved
- Priority: Low, Medium, High, Urgent
- Assignment ke agent (manual atau auto-routing)
- SLA rules per priority level
- Ticket queue dengan priority ordering


4. MULTI-CHANNEL SUPPORT
--------------------------
- Web Chat: Real-time via WebSocket
- Email: Gmail SMTP integration
- WhatsApp: Meta Cloud API integration (code ready)
- Semua channel masuk ke satu dashboard


5. ADMIN DASHBOARD
-------------------
- Overview: Statistik ringkasan
- Chat: Sub-menu Email (active), WhatsApp (active),
        Instagram/Inbox/WA Group (coming soon)
- Team: Submenu per user group (role), lihat member, profile modal
- Tickets: Filter by status
  (All, Unassigned, Pending Investigator/Programmer, Closed, Resolved)

- Knowledge: Upload, manage, rebuild KB

            Macros, Audit Logs

- Agent bisa takeover dari AI

- Provider: Google Vertex AI

- Fungsi: Mengubah dokumen KB menjadi vector untuk semantic search
- Dimensi: 768-dimensional vectors
- Disimpan di: FAISS index (data/db_storage/index.faiss)

- Provider: Google Vertex AI
- Model: Gemini 2.5 Flash
- Region: asia-southeast1 (Singapore)
- Temperature: 0.3
- Fungsi: Generate jawaban berdasarkan context dari KB

RETRIEVAL METHOD:
- Hybrid Search = BM25 (keyword) + FAISS (semantic)
- BM25 weight: 40%
- Vector weight: 60%
- Merge method: Reciprocal Rank Fusion (RRF)

LLM FALLBACK CHAIN:
  Vertex AI (Gemini 2.5 Flash)
    -> Google Gemini API (direct, via API key)
      -> Groq (free tier)
        -> OpenAI (GPT-4o-mini)

### Comprehensive Review and Enhancement Strategy for Edgeworks Support Portal

The Edgeworks Support Portal represents a solid, AI-powered replacement for legacy systems like Freshdesk, tailored to PT Edgeworks Solutions Indonesia's POS offerings for restaurants and retail. Built on a modern tech stack including FastAPI, Supabase PostgreSQL, and Google Vertex AI's Gemini 2.5 Flash model, it automates customer inquiries through a RAG (Retrieval-Augmented Generation) system, manages tickets, and supports multi-channel interactions. This review draws on the provided project details, evaluating its architecture, features, and deployment against industry best practices for AI customer support portals. It highlights strengths while proposing targeted improvements to achieve world-class status—defined by scalability, security, user-centric personalization, and seamless integration, as seen in leading platforms like Zendesk, Intercom, and Salesforce Einstein Service Cloud.

World-class AI support systems, per analyses from sources like Kustomer and Nextiva, prioritize not just automation but also proactive engagement, data-driven insights, and robust security to handle millions of interactions efficiently. The Edgeworks portal already excels in core areas like hybrid search and RBAC (Role-Based Access Control), but opportunities exist to enhance performance, especially for production-scale growth. Below, we break down the review into key components, followed by a detailed improvement roadmap with logical rationale grounded in benchmarks and real-world examples.

#### Architecture and Tech Stack Evaluation
The portal's architecture is modular and efficient: a FastAPI backend with async capabilities handles REST and WebSocket endpoints, supported by services for RAG, ticketing, and escalation. Frontend uses simple Jinja2 templates with Tailwind CSS, keeping it lightweight. Database (Supabase PostgreSQL) and vector store (local FAISS with BM25) enable quick retrieval, while integrations like WhatsApp Meta API add omnichannel appeal.

**Strengths**:
- **Async Efficiency**: FastAPI's use with Gunicorn/Uvicorn workers supports concurrent requests well, ideal for real-time chat.
- **Hybrid Retrieval**: Combining BM25 (keyword) and FAISS (semantic) with RRF (Reciprocal Rank Fusion) at 40/60 weights improves accuracy over pure vector search, aligning with best practices from LangChain for RAG systems.
- **Cost-Effective Deployment**: Docker on GCP e2-small keeps costs low (~$20-50/month), with Supabase for managed DB reducing ops overhead.

**Limitations**:
- **Scalability Bottlenecks**: Local FAISS storage on disk lacks distributed scaling; high-traffic spikes could cause latency or downtime.
- **Observability Gaps**: While custom logging exists, optional Langfuse integration isn't mandatory, potentially missing LLM tracing in production.
- **Dependency Management**: 71 packages introduce vulnerability risks; no mention of automated dependency scanning.

In comparison to world-class systems:
- Zendesk uses autonomous AI agents for 80%+ automation, far beyond the portal's confidence-based escalation.
- Intercom employs microservices for seamless scaling, suggesting a shift from the current monolithic FastAPI app.

#### Feature Set Analysis
The portal covers essential AI chatbot, knowledge base, ticketing, multi-channel, and admin dashboard features. AI handles greetings and low-confidence queries with disclaimers, escalating to agents via WebSockets. This mirrors best practices from Atlassian and IBM, where AI acts as a "first line" for routine tasks.

**Strengths**:
- **RAG Implementation**: Chunking/embedding with text-embedding-005 and Gemini generation ensures context-aware responses in Indonesian, with "Kak" for friendly tone.
- **Multi-Channel Unity**: Web, email, and WhatsApp feed into one dashboard, reducing silos.
- **SLA and Routing**: Auto-intent detection and priority-based queues enhance efficiency.

**Limitations**:
- **Lacks Advanced AI**: No sentiment analysis or proactive alerts, unlike Kustomer's tools that predict issues from chat patterns.
- **Personalization Deficit**: Responses aren't tailored by customer history or behavior, missing opportunities for upselling POS features.
- **Analytics Shortfalls**: CSAT surveys exist, but no deep insights like agent performance metrics or churn prediction.

Real-world benchmarks: Amtrak's AI "Julie" reduced handling times by 87% with similar RAG, indicating potential for Edgeworks to achieve 50-70% automation gains with enhancements.

#### Security and Compliance Review

**Strengths**:

**Limitations**:

#### Improvement Roadmap: Making It World-Class
To transform the portal into a world-class system, focus on scalability, advanced AI, security hardening, and analytics. Each suggestion includes logic, expected impact, and implementation steps, backed by industry data.

1. **Upgrade Vector Store to Managed Service (e.g., Pinecone or Weaviate)**  
   **Logic**: Local FAISS is fine for prototypes but struggles with production-scale indexing and queries (e.g., >1M vectors). Managed DBs like Pinecone offer serverless scaling, hybrid search out-of-the-box, and 99.9% uptime, per Towards AI and ZenML benchmarks. This reduces latency from 100ms+ to sub-50ms, improving RAG accuracy by 20-30%.  
   **Impact**: Handles 10x traffic growth; integrates seamlessly with LangChain.  
   **Steps**: Migrate embeddings to Pinecone (free tier for testing); update rag_service.py to use Pinecone client. Cost: ~$0.05/hour for starter pod.  
   **Benchmark Table**: Comparison of Vector DBs for RAG  

   | Database     | Scalability | Hybrid Search | Cost (per 1M vectors) | Best For          |
   |--------------|-------------|---------------|-----------------------|-------------------|
   | FAISS (Local)| Low        | Custom       | Free (disk-based)    | Prototypes       |
   | Pinecone    | High       | Built-in     | $0.10-0.50/hour     | Production RAG   |
   | Weaviate    | High       | GraphQL      | Free OSS; $0.20/hour | Feature-rich     |
   | Qdrant      | Medium-High| Excellent    | $0.05-0.30/hour     | Price-performance|
   | Milvus      | High       | Scalable     | Free OSS; variable   | High-volume      |

Proactive notifications (e.g., via WhatsApp for POS updates) mimic Delta Airlines' "Ask Delta" for higher engagement.

   **Impact**: Supports 10x users; reduces downtime.  
   **Steps**: Refactor services into Docker-compose micros; deploy to GKE. Use CI/CD with GitHub Actions.

5. **Add Analytics and Personalization**  
   **Logic**: Integrate tools like Google Analytics or Mixpanel for insights, enabling personalization (e.g., POS-specific recommendations) as in Sephora's chatbot (Virtu IT Systems). This aligns with EverAfter's AI personalization, improving retention.  
   **Impact**: 20% uplift in engagement.  
   **Steps**: Add endpoints for usage tracking; use ML in llm_service.py for tailored responses.
