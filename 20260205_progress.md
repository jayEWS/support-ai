## 1. Project Overview

**Support Portal Edgeworks** is a full-stack AI-powered customer support platform built for **PT Edgeworks Solutions Indonesia**. It provides multi-channel customer support (Web Portal, WhatsApp, Email) with an AI chatbot powered by RAG (Retrieval-Augmented Generation), real-time live chat between agents and customers, ticketing system, knowledge base management, and a comprehensive admin dashboard.

### Tech Stack

| Component | Technology |
|-----------|-----------|
| **Backend** | Python 3.11, FastAPI, Gunicorn + Uvicorn |
| **Database** | Supabase PostgreSQL (SQLAlchemy ORM) |
| **AI/LLM** | Vertex AI Gemini 2.5 Flash (with fallback chain: Gemini → Groq → OpenAI) |
| **Embeddings** | Vertex AI `text-embedding-005` / HuggingFace `all-MiniLM-L6-v2` |
| **Vector Store** | FAISS (local) with BM25 hybrid search |
| **Real-time** | WebSocket (FastAPI native) |
| **Frontend** | Jinja2 templates, Tailwind CSS, vanilla JavaScript (SPA-style) |
| **Deployment** | Docker on GCP VM (e2-small), Nginx reverse proxy, Let's Encrypt SSL |
| **Storage** | Google Cloud Storage (GCS) for knowledge file sync |
| **WhatsApp** | Meta Cloud API (pure, no Bird dependency) |
| **Email** | Gmail SMTP for notifications |

### 4.4 `app/core/state_machine.py` — Ticket State Machine

5-state ticket lifecycle with enforced transitions:

```
ON_HOLD (A-)          → PENDING_PROGRAMMER, PENDING_AGENT, CLOSED
PENDING_PROGRAMMER (B-) → PENDING_AGENT, CLOSED
PENDING_USER (C-)      → PENDING_AGENT, ON_HOLD, CLOSED
PENDING_AGENT (D-)     → PENDING_USER, PENDING_PROGRAMMER, ON_HOLD, CLOSED
CLOSED (E-)            → PENDING_AGENT (reopen)
```

| Function | Description |
|----------|-------------|
| `get_status_from_summary(summary)` | Map ticket summary prefix (A-/B-/C-/D-/E-) to status |
| `transition(ticket_id, current, next, agent_id)` | Validate and execute state transition; raises `ValueError` on invalid |
